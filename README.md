# Nossas Metas

Flutter project scaffolded with the official Flutter App Architecture guidance,
using MVVM in the UI layer and repositories/services in the data layer.

## Architecture

- `lib/ui`: Views and ChangeNotifier-based ViewModels.
- `lib/data`: Repositories and services that hide transport details.
- `lib/domain`: App models shared across layers.
- `lib/core`: Cross-cutting infrastructure such as DI, network, logging and errors.
- `lib/config`: App-wide configuration.
- `lib/routing`: Native named route definitions and router.

## Tooling

The project uses `provider` for dependency injection and state management,
Firebase Firestore for persistence, `flutter_lints` through
`analysis_options.yaml`, and examples for unit and widget tests.

Run:

```sh
flutter pub get
flutter analyze
flutter test
```

## Firebase

The web app is configured through `lib/firebase_options.dart`, generated by
FlutterFire CLI for the Firebase project `gto-eccomercce`.

```sh
flutter run -d chrome
```

Publish Firestore rules with:

```sh
firebase deploy --only firestore:rules
```

Because the product intentionally has no authentication, anyone with access to
the app can read and write tasks.
